---

## Decision: stepper vs timeline vs tabs

| Component | Role |
|---|---|
| `stepper` | Linear wizard progress. User moves through sequential steps. |
| `timeline` | Chronological event history. Past events, no navigation. |
| `tabs` | Non-linear panel switching. No implied sequence. |

Use stepper when: the user is in the middle of a multi-step process and needs to see where they are and how many steps remain (approval workflows, onboarding wizards, document publication flow).

---

## HTML pattern

```html
<ol data-ln-stepper>
	<li data-ln-step="complete">
		<span data-ln-step-label>Draft</span>
	</li>
	<li data-ln-step="complete">
		<span data-ln-step-label>In Review</span>
	</li>
	<li data-ln-step="current" aria-current="step">
		<span data-ln-step-label>Approval</span>
	</li>
	<li>
		<span data-ln-step-label>Published</span>
	</li>
</ol>
```

---

## State attributes

| Attribute | Visual |
|---|---|
| `data-ln-step="complete"` | Filled primary circle; connector to next step also turns primary |
| `data-ln-step="current"` | Filled primary circle + 4px halo ring (`primary / 0.2`) |
| (none / upcoming) | Neutral-200 circle; neutral connector |

Step numbers are generated by CSS `counter(ln-step)` — do NOT put numbers in HTML.

---

## Accessibility

- Use `<ol>` (ordered list) to convey sequence to screen readers.
- Add `aria-current="step"` on the current `<li>` alongside `data-ln-step="current"`.
- Label text is inside `[data-ln-step-label]` — this is the accessible name for each step.
- Non-linear navigation (clicking back to a previous step) must be handled by JS; the mixin is display-only.
